Data Types

There are 8 datatypes in JavaScript

  1. Number

    2. Any number, including numbers with decimals: 4, 8, 1516, 23.42.

    JavaScript does not differentiate between floats and integers


  2. String

    3. Any grouping of characters on your keyboard (letters, numbers, spaces, symbols, etc.) surrounded by single quotes: ' ... ' or double quotes " ... ", though we prefer single quotes. Some people like to think of string as a fancy word for text.


  3. Boolean

    4. This data type only has two possible values— either true or false (without quotes). It’s helpful to think of booleans as on and off switches or as the answers to a “yes” or “no” question.


  4. Null

    5. This data type represents the intentional absence of a value, and is represented by the keyword null.


  5. Undefined

    6. This data type is denoted by the keyword undefined (without quotes). It also represents the absence of a value though it has a different use than null. undefined means that a given value does not exist.


  6. Symbol

    7. A newer feature to the language, symbols are unique identifiers, useful in more complex coding.


  7. Object

    8. Collections of related data.


  8. BigInt

    9. Numbers between 9007199254740991 +(2^53-1) & -9007199254740991 +(2^53-1) and be safely stored. But numbers outside of this range needs to be declared as a bigint, otherwise they will be rounded and lose precision.

    Else if allows for more than two outcomes. You can add as many else if statements as you want. The else if statement comes after the if statement and before the else statement.

The Switch keyword

Else if statements can become quite long and overwhelming, which is when we can use the switch keyword

before:

let athleteFinalPosition = 'first place';

if (athleteFinalPosition === 'first place') {
console.log('You get the gold medal!');
} else if (athleteFinalPosition === 'second place') {
console.log('You get the silver medal!');
} else if (athleteFinalPosition === 'third place') {
console.log ('You get the bronze medal!');
} else {
console.log('No medal awarded.');
}

After:

let athleteFinalPosition = 'first place';

switch (athleteFinalPosition) {
case 'first place':
console.log('You get the gold medal!');
break;
case 'second place':
console.log('You get the silver medal!');
break;
case 'third place':
console.log('You get the bronze medal!');
break;
default:
console.log('No medal awarded.');
break;
}

Break keywod will stop the remaining cases from being checked.

A way to simplify the syntax of an if else statement

}

After:

isNightTime ? console.log('Turn on the lights!')
: console.log('Turn off the lights!');

JavaScript loading

Defer vs async?

Adding the defer attribute to a script result in that script running after the HTML document has been parsed, useful if the script depends on the html or other scripts

or: The defer attribute of a tag is a boolean attribute used to indicate that the script can be loaded but not executed until after the HTML document is fully parsed.

Scripts are loaded synchronously as they appear in an HTML file, before the following HTML is loaded and parsed. The async attribute can be used to load the scripts asynchronously, such that they will load in the background without blocking the HTML parser from continuing.

The Document Object Model (DOM)

The DOM is a logical tree-like Model that organizes a web page’s HTML Document as an Object.

The DOM also allows access to the attributes of an HTML element such as style, id, etc.

The document object (document.) allows scripts to access children of the DOM as properties.

Select and Modify Elements

  • .querySelector()

    The .querySelector() method allows us to specify a CSS selector as a string and returns the first element that matches that selector.

  • .getElementById()

    To access an element directly by its id, you can use the aptly named .getElementById() method

  • .getElementsByClassName()

    Return an array of elements unless we specific which index we want by adding []

  • .getElementsByTagName()

    Return an array of elements unless we specific which index we want by adding []


The main difference between querySelector() and getElementsByTagName() is that querySelector() returns the first matching element it finds, while getElementsByClassName() returns a collection of elements (an HTMLCollection) that match the specified class name.

So, if you want to select the first element that matches a specific class name, you should use querySelector().

On the other hand, if you want to select all elements that match a specific class name, you should use getElementsByClassName().

Appending, removing and creating elements

You can create, append, and remove elements by using the methods:

  • .createElement() creates an element in memory
  • .appendChild() adds a child to an existing parent element in the DOM
  • .removeChild() removes a child from an existing parent element in the DOM
